Objective-C Runtime(二) : 释放对象

2021-05-29

上回说到创建对象，这回来说下释放对象。在iOS中，使用引用计数来管理对象内存。iOS5之前，使用手动引用计数MRC(Manual Reference Counting)来管理内存，后面又推出了使用更广泛的自动引用计数ARC(Automatic Reference Counting)。MRC需要手动调用retain、release等，ARC则不需要手动调用，编译器帮我们做了。

简而言之，持有对象(retain)引用计数+1，释放(release)引用计数-1，当引用计数变成0，那么这个对象会被系统回收(dealloc)，释放占用的内存空间。

下面重点分析retain 、release、 dealloc过程。

retain

会先判断是不是TaggedPointer(由系统管理)或者是空，如果是则直接返回obj，其它对象进入reatain流程：

核心代码逻辑：

  • 如果不是nonpointer isa，也就是指针型isa，则直接操作散列表SideTable，引用计数+1
  • 如果是nonpointer isa，也就是非指针型isa(一般都是这种，针对64位指针优化过的)，isa中的extra_rc+1，也就是引用计数+1
  • 判断extra_rc满了，则拿出当前引用计数的一半存入extra_rc，并且设置isa的has_sidetable_rc为1(true)
  • 将剩下的另一半引用计数存入SideTable

release

查看release源码：

跟retain一样，首先判断是不是TaggedPointer(由系统管理)或者是空，如果是则直接返回，其它对象进入release流程：

核心代码逻辑：

  • 如果不是nonpointer isa，则直接操作散列表SideTable，引用计数-1
  • 如果是nonpointer isa，isa中的extra_rc-1，也就是引用计数-1
  • 如果extra_rc不为0，则将减完的引用计数存回isa
  • 如果extra_rc为0，则判断has_sidetable_rc是否为true，如果false，则进入deallocate流程(发送dealloc消息)
  • 如果has_sidetable_rc为true，则进入underflow流程，取出SideTable中一半的引用计数后-1并存入extra_rc
